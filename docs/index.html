Developing a Well-Designed Application

Design is centered around the creative process. I have adapted my career experience to fit a new process of design in software engineering.

Inventory & Analysis

The first step is always to identify the problem, and take inventory of what is already known. Learn what the client wants, needs or dreams of to improve their business. Take account of what the client already has, or identify similarities to previous projects.

Conceptual Design

Lay out a quick "sketch" of the project at a high level. The pseudo-code. Identify areas that need clarification, as well as the team's strengths and weaknesses in order to proceed with efficiency.

Master Planning

Take a modular approach to the concept - break down the design into small pieces to work methodically through the project. Coordinate each step with the team to ensure all elements are built and implemented seamlessly.

Back in 10!

Calendar app focused on wellness

Back in 10! is a full stack planning application that emphasises wellness and personal and mental health using calendar-style events.

I built the application using a React template for the client-side, incorporating Bootstrap and Sass as the primary styling methods. Axios calls on frontend CRUD actions make API requests to an Express template using Express router to send responses, with MongoDB and Mongoose to manipulate the database.

  • Primary Tools: React.js, Express.js
  • Deployed Site
  • Client Repository

PlusOne

Social event application

PlusOne is an application that allows users to post events they are hosting or attending, and invite others or allow other users to RSVP that they are also attending. This project was completed with a team of three other developers, with the objective of learning to manage multiple branches and code conflicts, as well as learning to work in a team environment.

The project was built using a React template for the client-side and an Express template for the server-side. Axios calls control all CRUD actions, with Express Routers receiving requests from the client. Primary styling was completed with Bootstrap and Sass.

  • Primary Tools: React.js, Express.js
  • Deployed Site
  • Client Repository

Take a Hike!

Hike tracking application

Take a Hike! is a full stack application for logging hikes or exercise activities. This project was an exercise in performing CRUD actions, and learning the connections between the client side requests and server side responses. This was our first full stack application, built using an Express API template to handle our requests. I used HTML, CSS/Sass and JavaScript on the client side. I used jQuery to handle click and submit events. I also used Handlebars for styling and layout of the index request and to prevent cross-site scripting.

  • Primary Tools: JavaScript, Express.js, Handlebars
  • Deployed Site
  • Client Repository
